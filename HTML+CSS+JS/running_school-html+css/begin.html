CHAPTER 1
기억해 두어야 하는 웹사이트 기본

1-1 좋은 웹 디자인이란?

  • 디자인은 무엇을 전달하기 위한 수단이다.
  • 웹 사이트의 쉬운 사용(접근성)은 사용자의 목적, 연령층에 따라 다양한 방식으로 설계 가능하다.
  • 유용한 도구일수록 사용자에게 편리성을 제공한다. 웹 디자인도 그러한 방식으로 사람들의 불편한 문제들을 해소해준다.

1-2 다양한 종류의 사이트

  1. 기업 사이트: 회사 개요, 회사 제품 소개 등을 위한 사이트이다.
  2. 프로모션 사이트: 이벤트, 서비스성 정보 전달을 위해 만든 사이트이다. 타깃층이 좁은 것이 특징이다.
  3. 포트폴리오 사이트: 디자이너, 사진 작가 등이 작품 소개를 목적으로 짓는 사이트이다.
  4. 쇼핑 사이트: 상품 판매 또는 해외 사업 등을 목적으로 만든 사이트이다. 결제 시스템과 구매 유도 전략 등이 특징이다.
  5. 미디어 사이트: 최신 뉴스나 정보 공개 목적으로, 블로그나 회사 소개 형식으로 된 사이트이다.
  6. SNS(Social Network Service): 실시간 소통, 업로드 목적으로 만든 사이트(플랫폼)이다.

1-3 사용성이란?

  • 사용성(Usability)은 사용하기 쉬움이라는 성질이다.
    웹 사이트의 기능 향상, Stress 없는 사용 유도가 주요 방침이다. 사용자의 니즈와 원하는 정보로의 쉬운 접근성이 중요하다.
  • 쉽게 디자인하는 방법은 색의 대비와 명확성이다. 텍스트의 크기나 버튼 요소의 크기 등을 조절하여 구현할 수 있다.
  • 읽기 쉬운 글은 결론은 먼저, 전문 용어 피하기, 제목 키워드 활용 등이 있다.
  • 사용하기 쉬운 조작성은 빠른 동작과 편리한 구성 방식을 말한다.
  • 추가로 대비를 쉽게 확인하는 방법은 콘텐츠의 내용을 스크린샷으로 확인하는 방법이다.
    색상에 통일감이 있더라도 그레이스케일(회색톤)로 바꿔서 보면 대비가 낮아서 글씨가 잘 안보이는 경우가 있다.

1-4 웹사이트 구조

  1. 인터넷(Internet): 전세계의 컴퓨터들이 정보를 교환하는 시스템이다.
  2. 웹(Web): 정식명은 World Wide Web이다. 인터넷을 이용한 정보 전송 확인 시스템이다.
  3. 웹 페이지 구조: 웹 서버, 웹 클라이언트로 나뉜다.
    • 웹 서버(Web Server): 컴퓨터에서 정보 공개 및 파일 저장 등의 기능을 수행한다.
    • 웹 클라이언트(Web Client): 웹 서버에 정보를 요청하고 서버가 응답한 결과를 받는 쪽이다.
  4. URL: 웹 사이트에 도착하는 주소이다. URL을 사용자가 직접 입력하거나 하이퍼링크를 통해 특정 URL에 이동할 수 있다.

1-5 장치의 종류

  1. 단말기
    • iOS 장치: 아이패드, 아이폰, 맥북, 애플워치 등 애플이 만든 장치
    • 안드로이드 장치: 안드로이드폰, 스마트워치 등 안드로이드OS 기반 장치
    • 모바일 장치: 휴대용 전자 기기를 지칭함. 태블릿, 디지털카메라, 노트북 등
    • 스마트 장치: 애플리케이션 사용 단말기. 스마트폰, 태플릿 등
    • 웨어러블 디바이스: 애플워치와 같이 몸에 걸치는 용도로 사용됨
    • IoT 장치: 사물이 인터넷과 연결된 형태
  2. 주변 기기
    • USB 장치: 키보드, 마우스, 프린터, 모니터 등
    • 저장 장치: SD 메모리 카드, 하드 디스크 등
    • 오디오 장치: 스피커, 헤드셋 등 음성 입출력 장치

1-7 제작 흐름

1 기획 웹 사이트의 목적, 필요한 콘텐츠를 구상한다. 메인 타깃이 어떻게 사이트를 사용할지 정한다.
2 사이트 맵 만들기 사이트 맵은 웹 사이트의 구성 나타내기, 즉 각 페이지와 페이지 간 이동을 그림으로 정리하기이다.
3 와이어프레임 만들기 기본 도형으로 간략한 선과 박스를 만들고 필요한 항목, 우선순위, 배치 등을 확인한다.
4 디자인 본격적인 디자인 작업, 웹에 그대로 구현하도록 세부적 요소까지 완성한다.
5 코딩 디자인이 동작하는 형태, 이미지와 글 등 출력 결과를 설정한다.
6 공개 웹 서버에 업로드, 공개 후에는 재검토가 필요하다.
